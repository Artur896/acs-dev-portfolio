01
Producción E-commerce Pagos

Educación Médica Especializada

Plataforma de venta y gestión de cursos de especialización médica con pagos internacionales

🎯 El Problema

La empresa gestionaba inscripciones y pagos de forma manual — transferencias bancarias verificadas a mano, sin control de acceso automático al contenido tras el pago, y sin historial de transacciones consolidado.

⚙️ El Proceso

React
SPA
→
Laravel
API REST
→
Stripe / PayPal
Webhooks
  • Roles con Laravel Sanctum: Admin, Instructor y Alumno con middleware de acceso propio y rutas protegidas.
  • Webhooks de Stripe y PayPal: El acceso al curso se activa automáticamente al confirmar el pago, sin intervención humana.
  • Idempotency keys: Garantizan que los webhooks se procesen exactamente una vez, incluso con reintentos.

🔥 El Desafío

Evitar doble activación de accesos ante reintentos de webhooks. Implementé una tabla de eventos procesados y verificación de idempotency key antes de ejecutar cualquier acción.

📊 Resultados

Tiempo de activación de acceso
Antes: 24-48h (manual)
Después: <30s
Doble cobro en producción
0% registrado desde el deploy

03
Freelance E-commerce Inventario

Ferreteros Unidos

Plataforma de e-commerce y gestión de inventario para red de ferreterías — ventas online + control de stock en tiempo real

🎯 El Problema

La red de ferreterías operaba sin catálogo digital ni ventas online. El inventario se manejaba en hojas de cálculo desconectadas entre sucursales, generando quiebres de stock invisibles y pedidos duplicados.

⚙️ El Proceso

React
Catálogo
→
Laravel
API + Admin
→
MySQL
Inventario
  • Catálogo con búsqueda: Más de 2,000 productos indexados con filtros por categoría, marca y disponibilidad por sucursal.
  • Control de stock en tiempo real: Cada venta — online o en caja — descuenta automáticamente del inventario de la sucursal correspondiente.
  • Panel de administración: Gestión de productos, sucursales, pedidos y reportes de ventas con exportación a CSV/PDF.

🔥 El Desafío

Sincronizar el inventario entre tienda online y punto de venta físico sin conflictos de concurrencia. Usé transacciones de base de datos con bloqueos a nivel de fila para garantizar consistencia.

📊 Resultados

Quiebres de stock no detectados
Antes: frecuentes
Después: ~0
Canal de ventas
Online habilitado desde cero
Productos en catálogo
2,000+ SKUs gestionados
