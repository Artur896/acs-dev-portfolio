01
Freelance Web SaaS Educación Salud

Educación Médica Especializada

Plataforma de educación médica con panel de administración completo para gestionar cursos de certificación (CRIO, ALSO, BLSO) dirigidos a profesionales de la salud en México, con presencia en 16+ estados y más de 5,000 profesionales capacitados.

🎯 El Problema

La organización coordinaba cursos de certificación médica a nivel nacional sin una plataforma centralizada. Las inscripciones se gestionaban manualmente, los comprobantes de pago llegaban por vías informales, y no había forma de que los alumnos consultaran su historial, accedieran a materiales o participaran en foros entre sesiones. La escala nacional (32 estados disponibles) hacía inviable seguir operando sin un sistema propio.

⚙️ El Proceso

PHP + MySQLi/PDO
Backend + Admin
→
MariaDB
Datos
→
JS Vanilla + AJAX
Interactividad
  • Portal del alumno: Registro con verificación de correo (código de 6 dígitos, 30 min), login con email o Google OAuth, dashboard personal con cursos inscritos, foro por curso, galería de imágenes y syllabus en PDF.
  • Pasarela de pagos múltiple: Integración simultánea de Stripe, PayPal, OXXO y Amazon Pay, más flujo manual de comprobante fotográfico con soporte de hasta 12 archivos por transacción.
  • Panel de administración: Gestión completa de cursos, lista de alumnos con estado de inscripción, seguimiento de ventas por tipo de curso (CRIO/ALSO/BLSO), generación de reportes en Excel y registro de actividad del admin.
  • Campañas de correo masivo: Sistema de cola en base de datos procesada por cron job, con soporte de adjuntos, selección de destinatarios y seguimiento de progreso en tiempo real.
  • Webinars en vivo: Implementación WebRTC con acceso exclusivo de cámara/micrófono para el admin y sistema de comentarios en tiempo real para los asistentes.
  • Gestión de rosters: Sistema de estados (pendiente → perfil creado → datos confirmados) para el control de inscripciones por tipo de curso.

🔥 El Desafío

Construir un SaaS médico de alcance nacional sin frameworks modernos — PHP puro con consultas preparadas, JS vanilla y sin ORM. El mayor reto fue el sistema de pagos: coordinar cuatro pasarelas distintas (incluyendo OXXO, que requiere flujo asíncrono) junto a un proceso de verificación manual de comprobantes. La escala de correos masivos también obligó a diseñar un sistema de cola propio para evitar timeouts, con procesamiento en background vía cron.

Freelance Web Institucional Catálogo Digital

Ferreteros Unidos

Sitio web institucional con catálogo digital de 1,000+ productos y panel de administración para cadena de ferreterías con 50 años de experiencia en CDMX.

🎯 El Problema

La empresa operaba sin ninguna presencia digital. Clientes — incluyendo dependencias gubernamentales como Pemex, SEDENA, SAT y más de 16 alcaldías — debían llamar o visitar la tienda para consultar productos. El catálogo se manejaba de forma completamente manual, sin posibilidad de mostrarlo en línea.

⚙️ El Proceso

PHP
Sitio + Admin
→
MySQL
Catálogo
→
JavaScript / AJAX
Búsqueda en tiempo real
  • Catálogo con búsqueda y filtros: 1,000+ productos organizados en 15 categorías (cemento, aceros, plomería, impermeabilizantes, etc.), con búsqueda en tiempo real vía AJAX y paginación por sesión con orden aleatorio.
  • Panel de administración: Gestión completa de productos — alta, edición, eliminación, activar/desactivar — con subida de imágenes y autenticación por sesión.
  • Sitio institucional: Secciones de nosotros, 2 sucursales, 16 marcas distribuidoras (Truper, Makita, Bosch, Dewalt, Cruz Azul…), galería de 22+ clientes gubernamentales y flota de transporte propia.

🔥 El Desafío

Construir un catálogo funcional y rápido de 1,000+ productos sin frameworks modernos, usando PHP puro con consultas preparadas y AJAX vanilla. El reto fue mantener la búsqueda ágil y la experiencia fluida para usuarios con conexiones variables, además de que el panel de admin fuera usable por personal no técnico.

04
Freelance Web E-commerce Catálogo Digital

Acuario Martinez

Landing page animada con catálogo digital y panel de administración para tienda especializada en peces ornamentales y accesorios para acuario.

🎯 El Problema

La tienda no tenía presencia digital. Los clientes no podían consultar productos disponibles sin visitar físicamente el local, y el proceso de reservas se hacía solo por llamada. Sin inventario en línea, sin gestión de pedidos y sin forma de mostrar el catálogo fuera del local.

⚙️ El Proceso

Supabase
Base de datos + Admin
→
HTML / CSS / JS
Interfaz + Animaciones
→
WhatsApp API
Reservas automáticas
  • Landing page animada: Pantalla de carga tipo acuario con llenado de agua, peces decorativos con animaciones independientes en el hero, burbujas flotantes ambientales y revelado por scroll.
  • Catálogo digital: Página separada con filtros por categoría, búsqueda en tiempo real, tarjetas de producto con imagen, precio y botón de reserva directa por WhatsApp.
  • Sistema de reservas: Flujo completo desde modal de producto hasta mensaje pre-armado en WhatsApp, con registro automático del pedido en la base de datos.
  • Panel de administración: Auth con Supabase, CRUD completo de productos y categorías, subida de imágenes a Supabase Storage, gestión de pedidos con cambio de estado (pendiente / confirmado / entregado / cancelado) y dashboard con estadísticas.

🔥 El Desafío

Construir una experiencia visual inmersiva — peces animados con dirección correcta, pantalla de llenado de agua, burbujas — usando solo HTML, CSS y JS vanilla sin ninguna librería de animación. El reto técnico mayor fue integrar Supabase como BaaS completo (DB + Auth + Storage) para eliminar la necesidad de servidor propio y mantener el costo de operación cercano a cero, mientras el panel de administración fuera usable por personal sin conocimientos técnicos.
